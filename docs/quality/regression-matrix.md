## 2) 실행 전략 (권장)

### P0
- CI 파이프라인의 **필수 통과 게이트**로 설정
- 실패 시 머지 차단
- API 레벨 + E2E 최소 세트 동시 유지

### P1
- 일일 배치(야간) + 릴리스 전 전체 실행
- 실패 시 릴리스 리스크로 분류, 핫픽스 우선순위 상향

### P2
- 주간 회귀 또는 릴리스 전 점검에 포함
- 사용자 불편도 기반으로 우선순위 조정

## 3) 자동화 구현 권장 분류

- **높음**: 인증 API 검증, 토큰 검증, 권한 경계(401/403), 채팅 정렬/카운트 계산
- **중간**: SSE 재연결/유실 방지(네트워크 제어 필요), 다중 탭/다중 디바이스 세션 정책
- **낮음**: (현재 범위 내 해당 없음) 수동 UX 체감 위주의 시나리오만 해당

## 5) 핵심 리스크 Top3

1. **SSE 재연결 시 이벤트 유실/중복(SSE-03, SSE-04)**
   - 실시간 채팅 제품의 신뢰성 핵심 축. 누락/중복은 즉시 사용자 신뢰 저하 및 데이터 정합성 문제로 연결.

2. **토큰/권한 경계 취약점(PERM-02, PERM-03, PERM-04, PERM-05)**
   - 보안 사고로 직결될 수 있는 영역. 특히 IDOR, JWT 위변조 허용, 폐기 토큰 재사용은 치명적.

3. **세션 만료/재발급 경계 처리 실패(SESS-02, SESS-03)**
   - 정상 사용자 대량 로그아웃(가용성 저하) 또는 반대로 만료 세션 허용(보안 저하) 두 방향 모두 비즈니스 임팩트가 큼.
